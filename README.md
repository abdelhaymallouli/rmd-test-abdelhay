# RMD Junior Web Integrator + SEO Assistant — Take-Home Test

Welcome, and thanks for applying to **Rhillane Marketing Digital (RMD)**. This repository is the take-home assessment for the **Junior Web Integrator + SEO Assistant** position.

The test has 4 sections. You will complete all of them and submit your work as a **public GitHub repository**.

---

## How to start

1. Click the green **"Use this template"** button at the top of this repo on GitHub.
2. Choose **"Create a new repository"**.
3. Set its visibility to **Public**. Name it `rmd-test-<your-name>` (lowercase, hyphens only). Example: `rmd-test-younes-amrani`.
4. Clone your new repo to your machine and start working through the four sections in order.

---

## What you will do

| Section                     | Folder                                                | What you produce                                                                                          |
| --------------------------- | ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| 1. Multiple-choice quiz     | [`01-qcm/`](./01-qcm/)                                | Edit `01-qcm/answers.json` with your answers to the 25 questions in `questions.md`.                       |
| 2. Figma design + WordPress integration | [`02-figma-wordpress/`](./02-figma-wordpress/) | Design a landing-page section in Figma, build it on a WordPress site you set up, and submit the raw `.fig` file + the live URL. |
| 3. SEO audit                | [`03-seo-audit/`](./03-seo-audit/)                    | Audit the page you just published. Submit a structured `audit.md`.                                        |
| 4. AI + automations         | [`04-ai-automations/`](./04-ai-automations/)          | A short practical AI prompt task + an automation sketch + one open-thinking question.                     |

Each section folder has its own `brief.md` with the detailed instructions and acceptance criteria. **Read the brief before you start the section.**

---

## Deadline

You have **5 calendar days** from the moment you receive the link to this test.

We do not reward speed for its own sake — use the full 5 days if you need to. We do penalize sloppy or rushed work.

---

## Submission

When everything is in place:

1. Make sure your repo is **public**.
2. Make sure all four `submission/` folders contain the files the corresponding `brief.md` asks for.
3. Submit your repository URL via the form linked in the email you received from us.

## Rules of the road

- This is **your** work. We expect you to use AI tools (it is literally tested in section 4), but you must understand and own everything you submit. Be ready to explain any choice during the follow-up interview.
- **Don't copy** another candidate's repo. We compare submissions and we will notice.
- **Use the full 5 days if you need them.** Late submissions are rejected unless you tell us in advance.
- **Ask questions if a brief is genuinely unclear.** Email the address in your invitation. We would rather answer a clarification than receive work based on a misunderstanding. We do not give hints on the answers themselves.
